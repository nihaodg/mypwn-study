函数调用约定

理解参数如何传递、栈如何管理，这是逆向和Pwn的基础。

重要
x64 Linux

System V AMD64 ABI

参数传递顺序
第1参数
RDI
第2参数
RSI
第3参数
RDX
第4参数
RCX
第5参数
R8
第6参数
R9
超过6个参数会使用栈传递
返回值
RAX 或 RDX:RAX (大数值)
调用者保存 (Caller-saved)
RAX, RCX, RDX, RSI, RDI, R8, R9, R10, R11
被调用者保存 (Callee-saved)
RBX, RBP, R12, R13, R14, R15, RSP

函数调用过程演示

调用者 (Caller)
main 函数
mov rdi, 10 ; arg1
mov rsi, 20 ; arg2
call func
; RAX = result
→
被调用者 (Callee)
func:
push rbp
mov rbp, rsp
; RDI=10, RSI=20
add rax, rdi
add rax, rsi
pop rbp
ret

常见汇编模式

函数序言 (Prologue)
push rbp ; 保存调用者栈底
mov rbp, rsp ; 建立新栈帧
sub rsp, 0x20 ; 分配局部变量空间
函数尾声 (Epilogue)
leave ; mov rsp, rbp; pop rbp
ret ; 返回
快速清零
xor rax, rax ; 比 mov rax, 0 更快
原因: XOR同寄存器只需要1字节，且利用CPU内部并行
获取栈地址
lea rax, [rbp-8] ; 获取局部变量地址
常用于传递指针参数
