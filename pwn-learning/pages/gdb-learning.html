GDB 调试学习

掌握GDB调试器常用命令，理解程序运行时的内部状态。

环境准备

安装GDB
apt install gdb
安装pwndbg插件（推荐）
git clone https://github.com/pwndbg/pwndbg
pwndbg 提供更好的堆、栈、寄存器可视化

基础命令

r / run 启动程序调试 
(gdb) run
# 或简写
(gdb) r

# 带参数运行
(gdb) r arg1 arg2

# 从文件读取输入
(gdb) r < input.txt
效果：启动程序，程序会执行直到断点或结束
b / break 设置断点 
(gdb) break main                  # 在main函数入口打断点
(gdb) break vuln                 # 在vuln函数打断点
(gdb) break *0x401156            # 在指定地址打断点
(gdb) break *0x401156            # 内存地址断点

# 条件断点
(gdb) break vuln if x==10       # 当x==10时中断
(gdb) break 10 if i>100         # 第10行且i>100时中断

# 查看断点
(gdb) info breakpoints
(gdb) delete 1                  # 删除1号断点
效果：程序运行到断点处会暂停，等待调试命令
n / next s / step 单步执行 
(gdb) n / next    # 执行下一行（不进入函数）
(gdb) s / step    # 执行下一行（会进入函数调用）
(gdb) ni / nexti  # 单步汇编指令（不进入）
(gdb) si / stepi  # 单步汇编指令（会进入）

# 继续执行
(gdb) c / continue  # 继续运行直到下一个断点
(gdb) finish        # 执行到当前函数返回
next vs step：next把函数调用当作一行，step会进入函数内部
p / print 查看变量值 
(gdb) print x              # 查看变量x的值
(gdb) print/x x            # 16进制显示
(gdb) print/d x            # 十进制显示
(gdb) print *ptr           # 解引用指针
(gdb) print *0x601010      # 查看地址内容
(gdb) print &var           # 查看变量地址
(gdb) print array@len       # 查看数组（len为长度）

# 打印寄存器
(gdb) print $rax
(gdb) print $rsp
(gdb) print $rbp

# 打印结构体
(gdb) print *struct_ptr
格式化：/x十六进制 /d十进制 /t二进制 /s字符串
x 查看内存 
(gdb) x/10x $rsp         # 从RSP显示10个32位十六进制值
(gdb) x/20x $rsp         # 显示更多
(gdb) x/s 0x601010       # 以字符串显示
(gdb) x/20x 0x7fffffffe3e0  # 从指定地址显示

# 格式说明: x/[数量][格式][单位] 地址
# 格式: x=十六进制, d=十进制, s=字符串, i=指令
# 单位: b=1字节, h=2字节, w=4字节, g=8字节

(gdb) x/8gx $rsp         # 显示8个64位值(g=8字节)
(gdb) x/16xb buf         # 显示buf处16个单字节
(gdb) x/4i $rip          # 反汇编RIP处4条指令
disas / disassemble 反汇编代码 
(gdb) disassemble          # 反汇编当前函数
(gdb) disassemble vuln     # 反汇编vuln函数
(gdb) disassemble 0x401000, 0x401100  # 指定地址范围

# 带源码模式（需要-g编译）
(gdb) disas /s
(gdb) disassemble /r 0x401000, 0x401100  # 显示原始字节

栈相关命令

bt / backtrace 查看调用栈 
(gdb) bt                   # 查看完整调用栈
(gdb) bt 10                # 只看前10层
(gdb) frame 2             # 切换到第2层栈帧

# 输出示例:
# #0  vuln () at vuln.c:5
# #1  0x0000000000401156 in main () at main.c:10
# #2  0x00007ffff7a2d090 in __libc_start_main ()
info frame 查看当前栈帧信息 
(gdb) info frame
# 输出:
Stack frame at 0x7fffffffe420:
 rip = 0x401156; saved rip = 0x4011a0
 called by frame at 0x7fffffffe430
 source language c.
 Arglist at 0x7fffffffe410, args: 
 Locals at 0x7fffffffe410, Previous frame's sp is 0x7fffffffe420
info registers 查看寄存器 
(gdb) info registers        # 显示所有寄存器
(gdb) info registers rax    # 只显示RAX
(gdb) info registers rsp rbp # 显示指定寄存器

# 输出:
rax            0x0     0
rbx            0x0     0
rcx            0x0     0
rdx            0x7ffff7fe180   140737488346496
rsi            0x7ffff7fe170   140737488346480
rdi            0x1     1
rbp            0x7fffffffe420   140737488350240
rsp            0x7fffffffe3f8   140737488350200
rip            0x401156   0x401156

内存与ELF命令

info 查看ELF信息 
(gdb) info functions        # 列出所有函数
(gdb) info functions vuln   # 查找vuln函数
(gdb) info variables        # 列出所有全局变量
(gdb) info sharedlibrary   # 查看加载的共享库

(gdb) info breakpoints     # 查看所有断点
(gdb) info watchpoints     # 查看监视点

(gdb) info proc mappings   # 查看进程内存映射
(gdb) info threads         # 查看线程
got / plt 查看GOT/PLT表 
(gdb) got                   # 查看GOT表
(gdb) plt                  # 查看PLT表

# 常用:
(gdb) x/10x got            # 查看GOT表内容
(gdb) x/10x plt            # 查看PLT表内容

# 找特定函数地址:
(gdb) got system           # system函数的GOT地址
search 搜索内存 
(gdb) search-pattern /bin/sh  # 搜索字符串
(gdb) find /bin/sh           # 搜索字节序列
(gdb) find $rsp, $rsp+100, 0x41  # 在栈上搜索0x41

Pwndbg 特有命令

注意：安装pwndbg后可用这些增强命令 git clone https://github.com/pwndbg/pwndbg && cd pwndbg && ./setup.sh
heap 查看堆状态 
(gdb) heap                   # 显示堆概览
(gdb) heapbins              # 显示所有bins状态
(gdb) fastbins               # 显示fastbins
(gdb) smallbins             # 显示smallbins
(gdb) unsortedbin           # 显示unsorted bin
(gdb) chunks                # 显示所有chunk详情
效果：可视化显示堆的bins链表结构
stack 查看栈内容 
(gdb) stack               # 显示栈内容（带注释）
(gdb) stack 30             # 显示30行栈
(gdb) stack -10           # 显示更多（向上）
(gdb) canary              # 查看栈canary值
vmmap 查看内存映射 
(gdb) vmmap                # 显示内存映射
# 输出:
# 0x00400000 0x00401000 r-xp /path/to/binary
# 0x00600000 0x00601000 rw-p /path/to/binary
# 0x7ffff7fc0000 0x7ffff7ffe000 r-xp [heap]
# ...
telescope 递归查看内存 
(gdb) telescope            # 查看地址内容并跟随指针
(gdb) telescope 20        # 查看20层
(gdb) telescope $rsp       # 从RSP开始查看
checksec 检查二进制保护 
(gdb) checksec
# 输出:
# [*] '/path/to/binary'
#     Arch:     amd64-64-little
#     RELRO:    Partial RELRO
#     Stack:    Canary found
#     NX:       NX enabled
#     PIE:      No PIE (0x400000)
ropsearch / search 搜索ROP gadgets 
(gdb) search pop rdi       # 搜索 "pop rdi"
(gdb) search pop rdx       # 搜索 "pop rdx"
(gdb) search "pop rdi; ret"  # 搜索完整gadget
got 查看GOT表（可视化） 
(gdb) got                 # 以表格形式显示GOT表
(gdb) got system          # 查找system的GOT条目

监视点 (Watchpoints)

watch 设置写监视点 
(gdb) watch x              # 当x被写入时暂停
(gdb) watch *0x601010      # 当地址被写入时暂停

# 设置读/写监视点
(gdb) awatch *0x601010     # 读或写时暂停
(gdb) rwatch *0x601010    # 读取时暂停

# 查看监视点
(gdb) info watchpoints
注意：硬件支持的监视点，数量有限

实战调试示例

调试栈溢出
(gdb) break gets              # 在gets函数入口打断点
(gdb) run                      # 运行程序

# 程序断在gets处
(gdb) x/10x $rsp               # 查看栈上内容
(gdb) print buf                 # 查看buf地址

# 单步执行
(gdb) ni                        # 执行gets
# 输入大量字符触发溢出

(gdb) x/20x $rsp               # 再次查看栈，观察溢出

# 查看返回地址
(gdb) print $rbp+8             # Saved RBP 上是返回地址
(gdb) x/x $rbp+8              # 查看返回地址内容
调试格式化字符串
(gdb) break printf
(gdb) run

# 程序断在printf处
(gdb) x/10x $rsp               # 查看栈参数
# 发现用户输入被当作格式字符串处理

(gdb) c                         # 继续
# 输入 %x.%x.%x 观察泄露

(gdb) x/s 0x601020             # 查看泄露的地址内容
调试堆溢出
(gdb) break malloc
(gdb) break free
(gdb) run

# malloc执行时
(gdb) heap                      # 查看堆状态
(gdb) x/10gx 0x602000          # 查看chunk内容

# free执行后
(gdb) fastbins                  # 查看fastbin链表
(gdb) x/10gx 0x602000          # 观察chunk被放入bins
完整调试流程
# 1. 启动调试
(gdb) file ./challenge          # 加载二进制
(gdb) checksec                  # 检查保护

# 2. 设置断点
(gdb) break main
(gdb) break vuln
(gdb) break *0x401156

# 3. 运行
(gdb) run < input.txt           # 带输入运行

# 4. 调试过程中
(gdb) backtrace                 # 查看调用栈
(gdb) info registers            # 查看寄存器
(gdb) x/20x $rsp               # 查看栈
(gdb) print variable           # 查看变量

# 5. 单步执行
(gdb) ni                        # 单步汇编
(gdb) si                        # 单步汇编（进入调用）
(gdb) finish                    # 执行到函数返回

# 6. 修改执行
(gdb) set $rax = 0x1234        # 修改寄存器
(gdb) set {int}0x601010 = 0    # 修改内存
(gdb) jump *0x401156            # 跳转到地址

# 7. 继续/退出
(gdb) continue                  # 继续运行
(gdb) detach                   # 分离调试
(gdb) quit                     # 退出gdb
